What is the Bootstrap Prompt?

A single, massive 1000+ line prompt that acts as a Master Orchestrator to jumpstart your project's AI infrastructure.

1. Discovery

Analyzes your repository (services, DBs, auth, infrastructure).

2. Documentation

Generates 9 canonical architecture markdown files.

3. Tool Config

Adapts the context for Cursor, Claude Code, and Antigravity.

So what is the prompt? It's a huge set of instructions—over 1200 lines—that you paste into an empty agent chat. It acts as a Master Orchestrator. First, it discovers how your project is built. Second, it documents that architecture. And third, it translates that documentation into the specific configuration formats needed by whatever AI tool you use.

Tool Configs (March 2026)

The prompt generates the specific folder structures required by modern AI tools.

Feature Cursor Claude Code Antigravity
Global Rules `.cursor/rules/project-context.mdc` (alwaysApply) `CLAUDE.md` (root, read automatically) Tool Memory Settings (manual)
Contextual Rules `.mdc` files activated via `globs` matching specific files Subdirectory `CLAUDE.md` files `.agent/memory/*.md` files (read on demand)
Workflows `.cursor/skills/*/SKILL.md` (Agent Skills) `.claude/commands/*.md` (Slash Commands) `.agent/workflows/*.md` (Slash Commands)
Project Index `AGENTS.md` Points to `AGENTS.md` if Combined Mode N/A (Managed by Knowledge Items/Subagents)

Note: We explicitly DO NOT use `.cursorrules` anymore. It is deprecated in favor of `AGENTS.md` and `.mdc` rule files.

This is the core technical value. In 2026, the way these tools manage context has splintered. Cursor uses the new `.mdc` file format and Agent Skills. Claude Code uses CLAUDE.md and slash commands. Antigravity uses its own memory and workflow `.md` files. Our prompt detects what tools you want and builds ONLY the adapter files for those specific tools, linking back to a single source of truth in the docs folder. Also, we kill the old .cursorrules file.

Rules vs. Skills vs. Agents

Rules Skills/Workflows Agents Subagents
Static context (e.g. `.mdc`). Constrains agent behavior via prompt injection. Actionable step-by-step procedures (e.g. Slash Commands). The autonomous brain executing Plan-Act-Reflect loops using tools. Child processes spawned by the main agent for parallel/specialized tasks.

Platform Capabilities

Platform Agent Paradigm Subagent Orchestration
Cursor IDE-Integrated Composer (Background file edits, terminal access). Asynchronous sub-agents natively supported (v2.5+), capable of spawning their own sub-agents and running dedicated VMs.
Claude Code Fast CLI Agent (file-by-file terminal execution, remote control). Supports Agent Teams (v4.6+) running up to 9 sub-agents in parallel for complex orchestration.
Antigravity Advanced Framework (Tool-heavy execution, UI generation). Deep multi-subagent support (native Browser, Knowledge, and Terminal subagents working in parallel).
It is crucial to differentiate these terms. Rules tell the agent HOW to behave. Skills tell the agent WHAT steps to follow for complex tasks. The Agent is the brain executing them. Subagents are specialized workers spawned by that brain. Recently, both Cursor and Claude have released major updates embracing sub-agents—Cursor now supports asynchronous sub-agents that can even run their own VMs, and Claude Code introduced Agent Teams. Antigravity continues to shine in deep orchestration with its specialized native subagents—like a dedicated browser-subagent to test a UI, or a knowledge-subagent to search past conversations.

Agentic Coding Workflows

Moving from autocomplete to autonomous orchestration.

How it works today

  • Plan: Breaks task into sub-tasks using LLM logic.
  • Act: Uses tools (Read, Write, Terminal, Search).
  • Reflect: Reads error logs, auto-corrects code.

Why Context Matters

  • An agent doing a multi-file refactor needs to know where the DB models live.
  • A custom /doc-cleanup workflow relies on knowing which docs are sacred.

Agentic workflows run on context. Fast, token-efficient context files = cheaper, smarter, faster AI agents.

Agentic workflows mean the AI isn't just finishing your sentence—it's executing a loop: Plan, Act, Reflect. To do this without destroying your app, the agent needs hyper-specific context. If you give an agent a workflow to "deploy to staging", it needs to know what your staging pipeline actually is. This prompt builds that precise context layer.

Example: The Doc-Cleanup Workflow

We generate a custom slash command/skill for the agent to maintain hygiene.


---
description: Audit and clean up project documentation
---
# Documentation Cleanup

## 1. Inventory
// turbo
List all `.md` files in `docs/` and project root with date/size.

## 2. Classify
Mark as Active, Stale (merge candidate), or Orphan (delete).

## 3. Consolidate & Archive
Merge Stale docs into `docs/architecture/`. Move Orphans to `.archive/`.
**Safety Rule:** Never auto-delete README, AGENTS.md, or architecture docs!
Here is an example of what the prompt builds. It generates a "Doc-Cleanup" skill for Cursor, or a slash command for Claude/Antigravity. This workflow tells the agent exactly how to inventory your docs, classify them, consolidate stale ones, and safely archive orphans. It turns the AI into your automated repo librarian.
